1
Chuẩn bị trước khi deploy
~2 phút

📋 Các thông tin cần có sẵn

a

Link GitHub repository

Copy URL của repo cần deploy. Ví dụ: https://github.com/username/my-app. Repo có thể là public (không cần auth) hoặc private (cần kết nối GitHub App trước).

b

Domain (tùy chọn hoặc tạo tự động)

Nếu có domain riêng: nhập vào. Nếu không: bấm nút "Tạo tự động" → hệ thống tạo subdomain miễn phí dạng {tên-app}-{hash}.matbaohosting.com

c

Chọn dự án & môi trường

Vào Dự án → chọn project muốn deploy → chọn môi trường (thường là production).

Lưu ý: Nếu repo là private, cần kết nối GitHub App trước ở Cài đặt → Git → GitHub.
2
Mở AI Auto Deploy Wizard
~10 giây
a

Vào trang project

Dashboard → Dự án → click vào project → chọn môi trường.

b

Bấm nút "AI Auto Deploy"

Ngay ở trang môi trường, tìm nút 🤖 AI Auto Deploy (thường ở dạng nút màu tím).

Wizard sẽ mở ra với 3 bước: Nhập đầu vào → Xem phân tích → Hoàn tất.
3
Chọn repository & Domain (Step 1/3)
~30 giây

2 chế độ nhập repo

a

Tab "Tài khoản GitHub" (đã kết nối)

Chọn tài khoản GitHub → chọn repository → chọn branch. Hỗ trợ private repos.

b

Tab "Dán link GitHub" (public repos)

Paste URL vào ô input, ví dụ: https://github.com/requarks/wiki. Không cần kết nối GitHub App trước.

Nhập Domain gốc (bắt buộc)

c

2 lựa chọn

Cách 1: Nhập domain có sẵn (ví dụ: my-company.com) → AI gợi ý subdomain app.my-company.com.

Cách 2: Bấm nút "⚡ Tạo tự động" → tạo subdomain miễn phí dạng {tên-repo}-{6-ký-tự}.matbaohosting.com.

d

Bấm "Phân tích Repository"

Nút này sẽ kích hoạt AI Analysis (bước tiếp theo).

Chú ý: Nếu domain trống, nút sẽ bị disabled. Phải nhập hoặc bấm "Tạo tự động" trước.
4
Phân tích Repository (AI làm việc)
~10-30 giây

AI thực hiện các việc sau

BướcHoạt độngKết quả
1Fetch files từ GitHubTải về Dockerfile, docker-compose.yml, .env, README.md, package.json...
2Detect stackNhận diện Node.js / Python / Go / Java / PHP / Ruby / .NET...
3Chọn chiến lượccompose > dockerfile > nixpacks (ưu tiên compose)
4Parse env varsTừ .env, .env.example, compose ${VAR}, README
5Detect well-knownWiki.js, Ghost, n8n... → gợi ý Docker image chính thức
6Sanitize compose8 bước: xóa nginx, convert ports, inject healthcheck, DB sync...
7Giải thích (AI)Output tiếng Việt về architecture, services, lưu ý
Tất cả đều deterministic: AI chỉ viết text giải thích. Các quyết định (strategy, config, ports) đều từ code analysis → an toàn, reproducible.
5
Xem kết quả & tùy chỉnh (Step 2/3)
~1-2 phút

Các thành phần hiển thị

a

Kết quả phân tích (4 sections)

📈 Tổng quan → 🔧 Services → ⚙ Cấu hình tự động → ⚠ Lưu ý

b

Badges hiển thị

Stack detected (ví dụ: Node.js), Strategy (Docker Compose), Well-known project (✨ Wiki.js).

c

Docker Compose preview

Hiển thị YAML đã sanitize. User có thể xem trước, không thể chỉnh sửa trên wizard.

d

Nếu là Well-Known Project

Chọn 📦 Docker Image chính thức (khuyến nghị) hoặc 🔨 Build từ source code. Phức tạp như Wiki.js nên chọn Docker image.

e

Domain Suggestions

AI giờ đã thông minh hơn: chỉ đề xuất domain cho services thực sự web-facing (bỏ qua DB, init containers, Selenium).

f

Bấm "Deploy ngay"

Hoặc "Quay lại" nếu cần sửa input.

Sau khi Deploy: Nếu có env vars trống (như API keys bên thứ 3), vào tab Môi trường → bấm 🧠 AI Smart Fill để AI gợi ý.

Deploy & theo dõi (Step 3/3)
~2-10 phút (tùy size)

Các việc diễn ra

Giai đoạnHoạt độngThời gian
1. CloneGit clone repo vào server~10-30s
2. BuildDocker build image (nếu cần build)~1-5 min
3. PullPull pre-built images (MySQL, Redis...)~30s-2min
4. StartKhởi tạo containers theo depends_on~30s
5. HealthcheckChờ DB sẵn sàng (service_healthy)~30s-1min

7. SSLCấp Let's Encrypt cert (nếu bật HTTPS)~30s

Theo dõi deployment

a

Tab "Triển khai" (Deployments)

Xem lịch sử deploy, click vào deployment để xem live logs.

b

Tab "Nhật ký" (Logs)

Xem container logs real-time. Chọn container từ dropdown nếu có nhiều services.

c

Khi deploy fail

AI tự động chuẩn đoán (Phase 23). Mở deployment lỗi → bấm 🧱 AI Chẩn Đoán → AI phân loại lỗi + gợi ý fix.

Nếu lỗi đơn giản: bấm 🔨 AI Auto-Fix → "Đồng ý" → AI sửa env vars → redeploy (tối đa 5 lần).

Owner có thể xem thống kê từ Dashboard → Thống kê AI Deploy: top lỗi phổ biến, top stacks, top repos fail nhiều.

🚨 Xử lý lỗi thường gặp

LỗiNguyên nhânCách fix
init-db exit 2 Image image non-root + volume owner root Đã auto-fix: user: "0:0" + set -e
ENCRYPTION_KEY required AI không generate secret cho env nhạy cảm Bấm AI Smart Fill trong tab Môi trường
network not found Network stale từ deploy trước Đã auto-fix: remove-then-create network
MYSQL_ROOT_PASSWORD missing Compose thiếu env bắt buộc cho DB Đã auto-fix: Phase 19 DB auto-fill
API key rỗng External credentials (ANTHROPIC_API_KEY...) User phải tự nhập từ nhà cung cấp
Build Wiki.js fail Wiki.js monorepo phức tạp Chọn Docker Image chính thức thay vì build source
Domain chưa có SSL Let's Encrypt cần DNS trỏ đúng Trỏ DNS về server trước, bật HTTPS sau
compose_*.yaml not found Compose có include: directive Đã auto-fix: Phase 31 detect include → clone full repo

✅ Checklist trước khi deploy

✅ Trước khi bấm Deploy
☐Repo GitHub tiếp cận được (public hoặc provider đã kết nối)
☐Domain gốc đã nhập hoặc bấm "Tạo tự động"
☐Xem trước Docker Compose (nếu có) → không có lỗi cú pháp
☐Đã chọn đúng Docker Image / Build from source (nếu well-known)
☐Kiểm tra env vars chấp nhận được (external API keys có thể nhập sau)
🔃 Sau khi deploy
☐Check tab Nhật ký → không có error lặp lại
☐Truy cập domain → app response đúng
☐Nếu có env external (API keys) → vào Môi trường nhập
☐Bật HTTPS sau khi DNS trỏ đến (Tab Tên miền)
☐(Nếu có) Setup backup cho database
